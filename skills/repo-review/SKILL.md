# Repo Peer Review

**원칙: 읽기만 한다. 파일을 절대 수정하지 않는다.**

---

## Step 1 — 프로젝트 구조 파악

다음을 한 번에 병렬로 실행한다.

```bash
find . -type f -not -path '*/node_modules/*' -not -path '*/.git/*' | sort
git log --oneline -10
```

`package.json` / `pubspec.yaml` / `Cargo.toml` 등 루트 설정 파일도 함께 읽는다.

파악할 것:

- 언어·프레임워크
- 주요 소스 디렉터리 (backend, frontend, ios-app, worker 등)
- 테스트 디렉터리
- 설정·환경변수 파일

---

## Step 2 — 핵심 파일 병렬 읽기

구조 파악 후 가장 중요한 파일 8–15개를 **한 번에** 읽는다.

우선순위:

1. 핵심 비즈니스 로직 (provider, service, route)
2. 데이터 정규화 / 중복 제거
3. 캐싱 레이어
4. 설정 / 환경변수 검증
5. 상태 관리 (ViewModel, store)
6. Worker / 백그라운드 동기화
7. 테스트 파일 1–2개 (테스트 패턴 파악용)

제외: 생성된 파일, 이미지·폰트 assets, lock 파일, 단순 SQL migration.

부족하면 추가로 읽되, 한 번에 묶어서 병렬로 읽는다.

---

## Step 3 — 이슈 분석

읽으면서 아래 카테고리를 기준으로 점검한다.

| 심각도 | 카테고리    | 주요 체크 포인트                                                        |
| ------ | ----------- | ----------------------------------------------------------------------- |
| 🔴     | 버그        | 중복 push/insert, 잘못된 slice 범위, 반전된 조건, null/undefined 오처리 |
| 🟡     | 성능        | O(n²) 루프 (find+indexOf 중첩), 불필요한 반복 API 호출                  |
| 🟡     | 메모리      | eviction 없는 모듈 수준 Map, 크기 제한 없는 배열                        |
| 🟠     | 로직 불일치 | 같은 함수의 null 처리가 파일마다 다름, 동일 기능 중복 구현              |
| 🟢     | 데드 코드   | 선언만 있고 어디서도 호출되지 않는 함수·메서드                          |
| 🔵     | 보안/설정   | CORS `origin: true`, API 키 노출 위험, 인증 없는 관리 엔드포인트        |

**포함 기준:** 실제로 발생할 수 있는 문제만. 린터·타입체커가 잡는 단순 스타일,
pre-existing 이슈(내가 읽은 코드 중 변경되지 않은 기존 문제), false positive는 제외.

---

## Step 4 — 출력 형식

### 파트 1: 이슈 목록

이슈가 있으면 심각도 순(🔴 → 🟡 → 🟠 → 🟢 → 🔵)으로 나열한다.
각 이슈는 다음 형식을 따른다.

```
### [심각도 이모지] [카테고리] #N — [한 줄 제목]
**파일:** `경로:라인번호`
**문제:** [2–3문장. 무엇이 왜 잘못됐는지, 어떤 상황에서 문제가 되는지]
```

### 파트 2: 통합 Codex 프롬프트

이슈가 1개 이상이면, 모든 이슈를 **하나의 프롬프트**로 묶어 코드블록으로 출력한다.
형식:

```
다음 파일들에서 발견된 이슈를 순서대로 수정해 줘.
수정 후 관련 테스트가 있다면 통과해야 하고, 없다면 추가해 줘.

---

## 1. [카테고리] [한 줄 제목]
파일: 경로 (라인 번호)

[현재 동작의 문제점 설명]
[기대하는 동작 또는 수정 방향]
[테스트 통과 조건이 있으면 명시]

---

## 2. ...
```

이슈가 없으면 파트 2를 생략하고 "이슈가 발견되지 않았습니다." 한 줄만 출력한다.
